# Welcome to Syntra public beta

Welcome to the Syntra community.

Syntra is an open-source, terminal-first control plane for coding with multiple
AI models. It can route planning, execution, and review separately, persist task
state on disk, and show the route, provider, and cost record behind a run.

## What this beta is for

Please use Syntra on real work and tell us where it is confusing, unreliable,
unsafe, expensive, or unexpectedly useful. The most helpful feedback includes:

- an installation/setup step that failed or was unclear;
- a route that selected an unsuitable model or provider;
- a cost, failover, permission, or verification outcome that surprised you;
- a missing provider/configuration example; or
- a focused contribution that improves docs, routing data, compatibility, or UX.

## Where to post

- **Q&A** — setup, commands, providers, and normal usage questions.
- **Ideas** — discuss a problem before proposing a large feature.
- **General** — contributor coordination and broader beta discussion.
- **Routing or provider feedback** issue form — routes, costs, model availability,
  catalog data, and failover behavior.
- **Show and tell** — share a sanitized workflow, integration, or project result.

Use GitHub Issues for reproducible bugs and defined feature work. Do not use a
public Discussion or Issue for a vulnerability: follow `SECURITY.md` and use
private vulnerability reporting.

## Keep reports safe

Never post API keys, OAuth tokens, provider config files, private code, customer
data, private paths, or `.syntra/` task state. Sanitized command output is welcome.

## Community expectations

Be specific, constructive, and respectful. Read `CODE_OF_CONDUCT.md` and
`CONTRIBUTING.md` before participating or sending a pull request.
