# Code of Conduct

Syntra should be a useful, respectful project for developers with different backgrounds and experience levels.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, but be clear when something is unsafe or incorrect.
- Keep feedback focused on the work, not the person.
- Help keep issues and pull requests actionable.
- Do not post secrets, private logs, or screenshots containing credentials.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing someone else's private information.
- Repeated low-effort noise after maintainers ask you to stop.
- Encouraging unsafe use of Syntra, such as leaking keys or bypassing safety gates.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject pull requests; or block users who violate these expectations.

If you need to report a concern, open a private security report if it is security-sensitive, or contact the maintainer through the GitHub profile linked from the repository.
